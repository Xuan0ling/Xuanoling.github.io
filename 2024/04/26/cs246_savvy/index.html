C++ Feature

Constent

Default Argument

default argument

  • when we define parameters for a function, we can assign a default balue for each parameter
  • when we make function calls, if such parameter is not passed as an argument, then its default value will be used
  • argument overwrites default value

Requirement in default arument

  1. Not every parameter needs a default value
  2. assume $p_1, p_2, p_3 … p_n$, a function with N parameters, if $p_k$ has a default value, then all parameters from $p_{k+1}$ to $p_n$ must have default value

Function Overloading

function signature

  1. function name
  2. number of parameters
  3. type of parameters
  4. sequence of parameters

three diffrent function calls:

  • goo(int);
  • goo(int, char);
  • goo(int, char, std::string);

number of default value + 1 is the total amount of function calls

with default argument, we are allowed to write functions with the same name, but function signature must be different

Tow stage:

  • compiling stage (error happened in this stage is called compiling eroor) must be static
  • runtime stage (error happend in this stage is called runtime error) must be dynamic

Function/Operator overload: static polymorphism (多态性)

  • static
  • dynamic

Struct in C++

Reference

Pass by Value (Pass by pointer is a special form)

Pass by reference

  • determined by parameter’s type
  • auto dereference
  • avoid copy

reference is an alias of an existing object

how to identify two variables are exactly the same
compare by memory address

pointer

  1. can be NULL
  2. can be initialized anytime (type and = can be in different line)
  3. can not point to other variables if needed
  4. need dereference to retrieve value

Reference

  1. cannot be NULL
  2. must be initialized upon creation
  3. reference cannot be changed
  4. automatic dereferenc
  5. similar as const pointer (int* const)

Lvalue

  1. appears on the both side of the assignment operator
  2. has memory address and is accessible (defined by us)
  3. by default, pass by reference is pass by LVALUE refrence
  4. const reference allows rvalue

Rvalue

  1. appears only on right side of the assignment operator
  2. does not have memory address (stored in register)
  3. && is for RVALUE reference
  4. if a function return by value, it always return RVALUE
  5. RVALUE is out-of-scope after use automatically
